# ü§ù Contribution Guidelines

Thanks for your interest in contributing!

Although this is a personal documentation repo, improvements are always appreciated.  
Please follow these guidelines if you'd like to contribute:

## 1. Keep things clear and concise

Write in a simple and direct way.  
Avoid unnecessary complexity or long paragraphs.

## 2. Follow the existing structure

Place your files in the correct folder.  
If unsure, open an issue to ask.

## 3. Use meaningful file names

Examples:

- `accessibility-overview.md`
- `frontend-performance-tips.md`

## 4. Keep tone consistent

Friendly, informative, and practical.

## 5. PRs & Issues

- Open an issue before making large changes.
- Pull requests should be focused on one topic.

Thanks for helping improve this knowledge base!
